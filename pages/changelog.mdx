## Breaking changes

Quando fazemos mudanças que quebram compatibilidade, seguimos este processo:

1. **Anúncio prévio**: Comunicamos com 90 dias de antecedência
2. **Período de transição**: Mantemos ambas as versões funcionando
3. **Guia de migração**: Fornecemos documentação detalhada
4. **Suporte**: Oferecemos suporte durante a migração

### Histórico de breaking changes

Até o momento, não houve breaking changes na API v1.

## Como se manter atualizado

### Notificações

Para receber notificações sobre mudanças na API:

1. **Email**: Configure notificações no [Dashboard](https://app.upaybr.com)
2. **Webhook**: Subscribe ao evento `api.version.updated`
3. **RSS**: Assine nosso feed RSS (em breve)

### Versionamento da API

A URL da API inclui a versão:

```
https://upay-sistema-api.onrender.com/api/v1/...
```

Quando lançarmos a v2, a v1 continuará disponível:

```
https://upay-sistema-api.onrender.com/api/v1/...  # Versão antiga
https://upay-sistema-api.onrender.com/api/v2/...  # Nova versão
```

### Versionamento dos SDKs

Os SDKs seguem versionamento semântico:

- **Major** (1.x.x): Breaking changes
- **Minor** (x.1.x): Novos recursos (compatível)
- **Patch** (x.x.1): Correções de bugs

```bash
# JavaScript/TypeScript
npm install @upay/upay-js@latest

# Python
pip install upay-python --upgrade

# PHP
composer update upay/upay-php
```
